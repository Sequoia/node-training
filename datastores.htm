Datastores and ORMs

  • MongoDB & Mongo
  • MariaDB (MySQL) & Sequelize
- Which to use - Look at mongo - Look at mysql

MongoDB

  • "Document database"
  • Schemaless
  • BSON
  • Relatively New
- documents look like JS objects mostly - no fixed structure for data - not text in/out like SQL - Leave data logic to application

Considerations

  • Easier to scale across servers
  • Documents are like JS Objects
  • Schemaless
  • No "Normalization"
  • Non-relational
- shards are a core concept (do you need this?) - shove JS objects directly into mongo - Flexible data models (pro and con) - no concept of normalization - non-relational, long lists of updates - *personally would not recommend as "go-to"

Where to get it

  • Package managers
  • https://mongodb.org/downloads
  • https://mongolab.com/

MariaDB (or MySQL)

  • SQL
  • Relational/has joins
  • Transactions
  • Very Mature

Considerations

  • Scaling more complex task
  • Tables/Schemas
  • Validation in data-store

Where to get it

  • Package managers
  • https://mariadb.org/download/
